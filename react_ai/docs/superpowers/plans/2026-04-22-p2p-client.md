# P2PClient Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the three exported helper functions in `src/services/api/p2p.ts` with a `P2PClient` class that owns the channel lifecycle, offers a pub/sub API over JSON envelopes, auto-responds to pings, and optionally maintains a peer registry in server mode.

**Architecture:** Single class in `src/services/api/p2p.ts`. One Loader `joinChannel` callback per instance acts as the sole dispatcher; it parses JSON envelopes, filters self-echo, updates the peer registry (server mode only), emits auto-pong for incoming pings, then fans out to user-subscribed handlers by `type`. Server mode adds a 60-second `setInterval` heartbeat that broadcasts `ping` and evaluates peer online status 3 seconds later.

**Tech Stack:** TypeScript, Vite, `window.Loader` (mtemplate-loader). No test framework is configured in this project — verification in this plan uses `tsc -b`, `oxlint`, and manual dev-server inspection described in each task.

**Spec reference:** `docs/superpowers/specs/2026-04-22-p2p-client-design.md`

**Existing callers of the old API:** a project-wide grep confirmed that nothing in `src/` imports `joinChannel`, `sendChannelMessage`, `sendChannelMessageJson`, or `ChannelMessageCallback` from `p2p.ts`. Removing the old exports is safe.

## Task 2: Wire up `joinChannel` with a named-callback dispatcher

**Files:**
- Modify: `src/services/api/p2p.ts`

- [ ] **Step 1: Add the dispatcher registration inside the constructor**

Replace the constructor body and add the private `dispatch` method. The callback given to `joinChannel` must have `callback.name === callbackName` because the Android player looks it up by function name; the computed-property-name trick below guarantees that.

```ts
constructor(channelName: string, clientId: string, isServer: boolean = false) {
  this.channelName = channelName;
  this.clientId = clientId;
  this.isServer = isServer;

  const callbackName = `__p2pCb_${++p2pCallbackCounter}`;
  // Computed property name sets callback.name === callbackName.
  // Required by the Android player which looks up the callback by function name.
  const named = {
    [callbackName]: (senderId: string, _channel: string, payload: string): void => {
      this.dispatch(senderId, payload);
    },
  };
  const callback = named[callbackName];
  Object.defineProperty(window, callbackName, { value: callback, configurable: true });
  getLoader().joinChannel(this.clientId, this.channelName, (window as never)[callbackName]);
}

private dispatch(_senderId: string, _payload: string): void {
  // filled in by later tasks
}
```

## Task 5: Implement the dispatcher body

This task fills in `dispatch` with parsing, echo filter, peer-registry update (server mode only), auto-pong, and fan-out.

**Files:**
- Modify: `src/services/api/p2p.ts`

- [ ] **Step 1: Replace the `dispatch` method**

```ts
private dispatch(_senderId: string, payload: string): void {
  let envelope: Envelope;
  try {
    const parsed = JSON.parse(payload) as unknown;
    if (
      !parsed ||
      typeof parsed !== "object" ||
      typeof (parsed as Envelope).type !== "string" ||
      typeof (parsed as Envelope).clientId !== "string"
    ) {
      console.warn("P2PClient: malformed message", payload);
      return;
    }
    envelope = parsed as Envelope;
  } catch {
    console.warn("P2PClient: malformed message", payload);
    return;
  }

  if (envelope.clientId === this.clientId) return;

  if (this.isServer) {
    const existing = this.peers.get(envelope.clientId);
    if (existing) {
      existing.online = true;
      existing.lastSeen = Date.now();
    } else {
      this.peers.set(envelope.clientId, {
        clientId: envelope.clientId,
        online: true,
        lastSeen: Date.now(),
      });
    }
  }

  if (envelope.type === "ping") {
    this.emit("pong");
  }

  const list = this.handlers.get(envelope.type);
  if (!list) return;
  for (const handler of list) {
    try {
      handler(envelope.data, envelope.clientId);
    } catch (err) {
      console.error("P2PClient: handler threw", err);
    }
  }
}
```

## Task 6: Implement server-mode heartbeat and `getPeers`

**Files:**
- Modify: `src/services/api/p2p.ts`

- [ ] **Step 1: Add `tick` and `evaluate` private methods**

Add just above `dispatch`:

```ts
private tick(): void {
  const pingTime = Date.now();
  this.emit("ping");
  setTimeout(() => this.evaluate(pingTime), 3_000);
}

private evaluate(pingTime: number): void {
  for (const peer of this.peers.values()) {
    peer.online = peer.lastSeen >= pingTime;
  }
}
```

- [ ] **Step 2: Start the heartbeat at the end of the constructor**

The constructor already calls `this.emit("ping")` as the initial ping (Task 3). Immediately after that line, schedule the first evaluation and start the interval — only when `isServer`:

```ts
this.emit("ping");

if (this.isServer) {
  const initialPingTime = Date.now();
  setTimeout(() => this.evaluate(initialPingTime), 3_000);
  setInterval(() => this.tick(), 60_000);
}
```

Note: `Date.now()` is called immediately after `emit("ping")` for `initialPingTime`; peers whose messages arrive before this capture will still satisfy `lastSeen >= initialPingTime` within the typical millisecond window.

- [ ] **Step 3: Replace the `getPeers` stub**

```ts
getPeers(): Peer[] {
  return Array.from(this.peers.values()).map((p) => ({ ...p }));
}
```

`.map(p => ({...p}))` copies each entry so callers cannot mutate internal state.

### P2PClient

`src/services/api/p2p.ts` exports a `P2PClient` class with a pub/sub API.

```ts
const p2p = new P2PClient("my-channel", "Device-A", true /* isServer */);
p2p.on("score", (data, from) => console.log(from, data));
p2p.emit("score", { points: 42 });
const peers = p2p.getPeers(); // server mode only
```

- Messages are JSON envelopes `{ type, clientId, data? }`. `clientId` is the device name.
- On construction the client joins the channel and broadcasts `ping`; on any incoming `ping` it replies with `pong` automatically.
- `"ping"` and `"pong"` are reserved type names but still delivered to user subscriptions if present.
- Server mode (`isServer: true`) broadcasts `ping` every 60 seconds and marks a peer `online: false` if it did not respond within 3 seconds of the ping. Peers are never removed from the list.
- No `off`/`once`/`dispose` — instances live for the template lifetime.
```
